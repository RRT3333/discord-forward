# 운영 당면 과제 (Open Items)

아래 항목은 현재 운영 안정성을 위해 우선순위로 관리할 과제입니다.

## P0 (즉시)

- Discord webhook 노출 이력이 있는 경우 즉시 폐기/재발급
- 429 재시도 실패(최대 재시도 초과) 시 에러 로깅/알림 채널 연동
- 실제 트래픽에서 429 발생 빈도 측정 (일/시간 단위)

## P1 (가까운 시일)

- 웹훅 전송 큐(직렬화) 도입
  - 목적: 버스트 상황에서 429 발생 최소화
  - 기준: 2초 창 요청량이 잦게 초과되는 경우
- 메시지 전송 실패 시 fallback 채널 또는 백업 웹훅 추가
- 본문 첨부 파일명/요약 포맷 표준화

## P2 (개선)

- 인코딩 품질 개선
  - 현재 `charset`/MIME encoded-word 복원 적용
  - 비표준 인코딩(일부 legacy 메일) 대응 여부 샘플 수집 후 추가 보완
- 운영 관측성 강화
  - Workers Logs 기반 429/forward 실패/파싱 실패 카운트 대시보드화
- 보안 운영
  - 시크릿 로테이션 주기 정의
  - 커밋 전 시크릿 스캔(gitleaks/trufflehog) CI 도입

## 운영 체크 지표

- Discord 429 비율
- Discord 전송 실패율
- 평균 메일 처리 지연
- 본문 파싱 실패율
- 한글 깨짐 신고 건수
